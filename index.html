JavaScript Information

JavaScript is a interpreted language, means it check top to bottom line by line and it compiled just-in-time. On the another side, JavaScript is a client side language. So, every browser has a engine to run JavaScript code (v8 engine, Spidermonkey engine etc). JavaScript engine has a parser, when it get JavaScript code, it parse to make a Syntax Tree and then it start it interpreter ignition . JavaScript is a high level language. So, after interpreter ignition it will do some process such as JIT complier and convert it into a Bytecode/Machine code. so that computer can understand those code.

JavaScript has a runtime model for running code. It has two cells to execute the code one is Stack and another is Queue. "Stack" maintain synchronous elements and listed code synchronous way to run line by line. On the other side Queue run asynchronous elements look like a waiting list. Unnecessary code send into garbage collection. Event Loop sets priorities and lists which code will be next? It does this process through loops so it called event loop.

local storage and session storage both are allow to save data in browser memory. But local storage save data for long time/permanently and session storage save data temporarily. When a site/page or browser close its session storage will be clear but in local storage data is saved until the browser uninstall. session storage data loosed after browser/site closed and local storage data save browser for long time.The main difference between the two is how long they can store data
